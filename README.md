# 口袋棋兵 Pocket Pawns

> 最後更新：2026-03-04

以六角格（Axial Hex）為舞台的輕量策略對戰遊戲。玩家操控一名角色，在充滿屬性格、干擾型 NPC（Knocker）以及對戰型 NPC（Combatant）的棋盤上求生存。

---

## 快速上手

1. 用靜態伺服器（或支援 ES Module 的瀏覽器）開啟 `index.html`。
2. 在右側面板選擇角色，按「開始遊戲」。
3. 點選相鄰六角格移動。
4. 點選自己角色會彈出技能選單。

> **注意**：因使用 ES Modules，需透過 HTTP 伺服器開啟（直接雙擊 `file://` 會 CORS 報錯）。
> 本機快速啟動：`npx serve .` 或 VS Code Live Server。

## 遊戲規則摘要

| 元素 | 說明 |
|------|------|
| **屬性格** | 每格帶有 `+` 或 `-` 標記的屬性（STR / AGL / VIT / DEX / LUK / INT）。站在格子上遭遇時，對應屬性以「比大」或「比小」比較雙方數值，平手時用 LUK 加權骰決勝。 |
| **Combatant NPC** | 其餘角色作為對手，各自隨機分配一種人格，每回合依人格 AI 行動。遭遇勝即獲得場數。 |
| **Knocker（干擾單位）** | 舞台上額外出現的彈飛機構，每回合在**環境階段**移動並將接觸到的單位撞向指定方向，可觸發連鎖出界或屬性格判定。 |
| **縮圈** | 隨回合推進棋盤有效範圍逐漸縮減，邊緣外的單位將被淘汰。 |

## 玩家技能系統（8 種）

| 圖示 | 技能名 | 效果 | 冷卻參考 |
|------|--------|------|----------|
| ⏸ | **待機** | 原地不動，消耗一個回合 | 固定 CD 3 |
| ⚡ | **衝刺** | 直線衝 2 格；需點選方向格確認 | AGL 越高 CD 越短 |
| 🗡 | **突破衝撞** | 自動朝最近敵人衝 2 格（受阻則退為 1 格） | STR 越高 CD 越短 |
| 🛡 | **鐵甲反彈** | 原地待機，下次被彈飛時方向強制反轉 | VIT 越高 CD 越短 |
| 🎯 | **快手連擊** | 移動 1 格，同時其他所有技能 CD 各 -1 | DEX 越高 CD 越短 |
| 🎲 | **亂入傳送** | 瞬間傳送至隨機空格 | 固定 CD 4 |
| 🎰 | **豪賭一擊** | 優先移動至相鄰負屬性格，LUK 加成判定 | LUK 越高 CD 越短 |
| 🔭 | **精算步伐** | 高亮距離 ≤ 2 的可選格，自由選定落點 | INT 越高 CD 越短 |

- 冷卻公式：`CD = max(1, N - floor(對應屬性 / 4))`
- 使用**快手連擊**後，本輪結束時所有其他正在冷卻的技能額外再 -1。
- `ESC` 可取消待選中的技能（衝刺方向選擇、精算步伐選格、快手連擊移動格）。

## NPC 人格系統（6 種）

Combatant NPC 與 Knocker 共用同一套 6 種人格，行為略有差異：

| 人格 | NPC 標籤 / Knocker 標籤 | 行為特色 | 技能（CD） |
|------|------------------------|----------|------------|
| **Aggressive** 勇猛型 | 勇 / AGG | 優先追擊最近單位 | 突破衝撞，強制彈飛 3 格（CD 3） |
| **Defensive** 謹慎型 | 守 / DEF | 偏好撞後安全落點，遠離縮圈邊緣 | 鐵甲反彈，目標往反方向彈飛（CD 4） |
| **Opportunist** 機會主義型 | 機 / OPP | 鎖定 VIT 最低對象，偏好把人撞向縮圈邊緣 | 連環追擊，彈飛 3 格（CD 3） |
| **Random** 隨性型 | 亂 / RND | 方向與目標全隨機，完全不可預測 | 狂亂衝刺，隨機方向衝 2 格（CD 2） |
| **Gambler** 賭徒型 | 賭 / GMB | 偏好負屬性格，偵向把目標撞到負屬性格 | 豪賭一擊，彈飛 4 格（CD 4） |
| **Tactician** 智將型 | 謀 / TAC | 計算連鎖效益，優先弱屬性格 + 多人聚集區 | 精算打擊，自動選最優彈飛方向（CD 3） |

### Combatant NPC

- 每名 NPC 隨機分配一種人格（6 種均可）。
- 每回合先判斷是否觸發技能（觸發後跳過一般 AI 邏輯），未觸發則走人格偏好移動。
- 觸發技能的回合，單位頭上顯示 `▶` 標示。
- 技能冷卻於玩家完成行動後遞減。

### Knocker 干擾單位

- 最多 8 隻，每隻分配唯一人格（6 隻以內保證不重複，超過才循環）。
- 每隻固定負責一種能力屬性（STR / AGL / VIT / DEX / LUK / INT）。
- 滑鼠停在 Knocker 上可查看人格名稱、說明、技能介紹與當前冷卻。
- 冷卻中的 Knocker 顯示 `CDx`，觸發技能回合顯示 `▶`。

## 音樂（BGM）

- 音樂檔放在 `sound/BGM/` 資料夾下，可循環播放多首。
- 頁面載入時嘗試自動播放；若被瀏覽器攔截，點擊任意區域解鎖。
- 面板音量滑桿與靜音按鈕，設定存於 `localStorage`（`ppf_bgm_volume`）。

---

## UI 面板功能

- **角色選擇**：5 名預設角色，各有不同屬性偏科。
- **棋盤背景**：無 / 格線 / 木紋 / 石材 / 專案圖片，支援上傳自訂背景。
- **HUD 資訊**：玩家座標、當前格屬性與規則、勝場數、待機冷卻、遊戲狀態。
- **判定記錄**：最近一次遭遇戰結果，含技能彈飛、鐵甲反彈等特殊訊息。
- **Knocker Tooltip**：滑鼠停在 Knocker 顯示人格說明與技能 CD 狀態。
- **重新亂數 / 鏡頭置中 / 重開一局** 快速按鈕。

## 專案結構

```
index.html              入口頁面
css/
  style.css             全站樣式
js/
  config.js             靜態常數、角色資料、人格定義
  state.js              全域可變狀態（player、State、圖片 Map）
  hex.js                六角座標工具函式
  board.js              棋盤生成、圖片載入
  ai.js                 NPC 放置、AI 行為、Knocker 干擾、環境彈飛階段
  combat.js             遭遇戰判定、淘汰邏輯、勝利檢查
  game.js               入口模組、回合控制、技能系統、事件監聽
  render.js             Canvas 繪製（格子、角色、NPC、Knocker、高亮）
  ui.js                 HUD 更新、Tooltip、技能選單
  bgm.js                BGM 播放控制
image/
  background/           棋盤背景圖片
  Knockback/            Knocker 人格圖片
sound/
  BGM/                  背景音樂檔案
docs/                   設計文件（SDD）
```

## 除錯提示

| 問題 | 排查方向 |
|------|----------|
| 角色圖片不顯示 | 確認 `image/` 下檔名大小寫正確，並透過 HTTP 伺服器開啟（非 `file://`） |
| Knocker 圖片不顯示 | 確認 `image/Knockback/` 下有 6 個 PNG，檔名須與人格英文名完全一致 |
| BGM 不播放 | 確認檔案在 `sound/BGM/`，或點擊頁面解鎖瀏覽器自動播放限制 |
| ES Module 錯誤 | 需透過靜態伺服器（如 `npx serve .` 或 VS Code Live Server）開啟 |
